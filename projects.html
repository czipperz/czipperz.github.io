A client / server app pair that connects Android phones to computers.

The feature I set out to implement when I created this app was the idea of "sharing" to your computer. This means that in YouTube or Chrome, you can share and select the app and it will open in the browser on your computer.

This app also allows for a "universal clipboard" whereby copying text on Android allows you to paste it on your computer and visa-versa.

In addition, you can share files and they will be placed on the computer in the Downloads folder.

Pairing is very easy: hit the plus button and then scan the QR Code shown on the server app.

A 4X game exploring lenses to see the game through.

This game is a 4X game (explore, expand, exploit, exterminate). Thus the user will start with a small city and build it up and defend it as time progresses.

One problem of this type of game is information overload and complexity of user commands. Thus the game aims to simplify this by exploring the idea of filtering data in the game. syrup

A C interpreter written in Haskell.

syrup is controlled through a REPL. This allows for machines to execute arbitrary programs instantaneously. In addition, syrup will parse files lazily for maximal performance. Currently it can query the type of a basic expression or evaluate it and print the result.

Both of these commands allow for C to become a much more powerful language. Type checking can become more reliable and simple when based on this software. Allowing C code to be executed dynamically nets C the ability to be used during a debugging process to better understand the state of the code, or even to modify it.

cutil

C utility library defining common data structures.

cutil implements various data structures in C, including a UTF8 string with short string optimization, a hash map, a hash set, a vector (dynamic array), a logging framework, and a cross platform dll loader.

Each of these components is thoroughly tested. Each of the data structures are generic to the type of the data without resorting to heap allocation of individual elements.

vick

Show more

A lightweight and modular text editor built in C++.

All components, including trivialities like movement, are built as plug-ins. This allows for the user to fully customize the experience by replacing any component at their own leisure.

In order to help manage these abundance of packages, I created a package manager that allows for users to send pull requests to register their own packages as well as local adjustment. It handles dependencies and versioning automatically.

In order to compile this machine, I also developed a custom build tool that automatically interfaces with the package manager and handles the entire process seamlessly.

Other plugins include insertion mode, joining two lines together, finding a character on the current line, predicates used for plugin development, compile commands for the user, and so on.

gcl

Show more

A better git clone.

gcl allows the user to type less to clone a new repository. For example, to clone gcl with itself, you can run gcl czipperz/gcl as it will automatically inject github.com into urls when deemed necessary.

It also allows the user to set up custom clone locations and access them later using a convenient syntax: gcl shortcut:path. c:https://github.com/czipperz/ in the configuration file will allow for using gcl c:gcl to clone gcl.

In addition, gcl allows the user to set up a username so that when it comes time to push, he or she will only have to type the password.

gr

Show more

A better git remote.

gr shortens the long commands of git remote into short ones. It does this similarly to how vim assigns key strokes to commands: acronyms. For example, gr ao url will run git remote add origin url.

It also shares the configuration file with gcl, allowing for url inputs to be automatically expanded when deemed necessary.
